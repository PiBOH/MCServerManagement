Minecraft server control panel

Gestione server Minecraft da un solo HTML

Un file piccolo ma Potente per gestire i tuoi server minecraft

N.B. Vedrai i tuoi server solo se usi ogni volta lo stesso browser e se non pulisci la cache di quest'ultimo, Ricordati di fare sempre un backup della serverlist e delle API

Regola tecnica non aggirabile

Da un file HTML il browser puo chiamare solo API che permettono CORS per origine null o per il dominio da cui apri la pagina. Se l'host non lo permette, nessun codice HTML puo bypassarlo senza proxy, estensione o app installata.

Server salvati

Non vengono creati server finti. Ogni pulsante invia una richiesta reale all'API configurata.

Nessun server aggiunto

Aggiungi un host con API ufficiale, per esempio un pannello Pterodactyl/Pelican o un endpoint API personalizzato fornito dal tuo hosting.

Compatibilita reale

Funziona aprendo il file su Windows e macOS se l'API dell'hosting:

  • accetta richieste da browser;
  • permette CORS per file:// oppure per il dominio usato;
  • accetta token via header, di solito Authorization: Bearer;
  • fornisce endpoint reali per start, stop e status.

Aternos e host senza API pubblica

Se l'host non offre una API ufficiale controllabile dal browser, questo file non puo avviare o spegnere il server senza usare un servizio intermedio. Per correttezza non ho aggiunto comandi finti.

Se dice che non riesce a connettersi

Non significa per forza che internet non funziona. Nei browser lo stesso errore appare anche quando l'host blocca CORS, rifiuta l'header Authorization o non risponde alla richiesta OPTIONS.

Errore proxy CORS

Se vedi un messaggio sui proxy CORS, non e un problema di AdBlock: significa che stai usando una versione/prova con proxy esterni oppure che l'host non permette chiamate API dal browser. Questa versione non usa proxy pubblici.

Cambiare browser non basta

Chrome, Edge, Vivaldi, Brave e Opera usano tutti Chromium e applicano le stesse regole CORS. Se falliscono tutti, il blocco e lato host/API, non lato PC.

Aggiungi server

Inserisci solo API/token reali forniti dal tuo host.

Pterodactyl / Pelican

Usa il Client API token del pannello. L'endpoint usato sara /api/client/servers/{id}/power.

API personalizzata

Configura gli endpoint esatti forniti dal tuo hosting. Il token resta nel tuo browser.

Sicurezza token

I token vengono salvati in localStorage del browser. Non condividere il file esportato se contiene chiavi API. Usa token con permessi minimi: solo power/start/stop/status se il pannello lo permette.
